Install locally with mock data

Spin up the quote experience for demos in minutes.

  1. Unzip or clone the starter and cd into it.
  2. npm install
  3. cp .env.example .env.local (defaults to DATA_PROVIDER=mock)
  4. npm run dev then open http://localhost:3000
  5. Try pages like /quote/%5EGSPC or /quote/AAPL

Switch to Polygon (optional)

  1. Get a Polygon API key.
  2. Set DATA_PROVIDER=polygon and POLYGON_API_KEY=your_key_here in .env.local.
  3. Restart npm run dev.
  4. Adjust index symbol mapping in lib/providers/polygon.ts if needed (e.g., ^GSPC → I:SPX).

Why it fits the live banking app

The quote page mirrors our product voice and transparency goals: clear price context, rich fundamentals, and actionable news in one place. Drop the Next.js app behind the Markets tab, and the rest of the experience (cards, deposits, payments) stays unchanged.

  • Design tokens in app/globals.css make it easy to align with your banking palette.
  • Route handlers are no-store today; add revalidation for SSR/ISR when you go live.
  • Options chain and index components are wired for mock data but ready for your production vendor.

Deployment checklist

  • Configure DATA_PROVIDER for your data source.
  • Confirm exchange/index symbol mappings per vendor.
  • Wire your CI (GitHub Actions provided) and container build (Dockerfile included).
  • Point DNS or reverse proxy to the Next.js app once verified.
